The RR/RRRR format (see section on RR/RRRR format within TIMESTAMP_FORMAT scalar function page) works slightly differently than YY/YYYY. The RR and RRRR format elements can be used to alter how a specification for a year is to be interpreted by adjusting the value to produce a 2-digit value or a 4-digit value depending on the leftmost two digits of the current year according to the following rules:

First two digits of the current year (ZZ) Last two digits of the current year (XX)Two-digit year in string-expression (YY)Resulting year
0-990-500-49ZZYY
0-9951-990-49(ZZ+1)YY
0-990-5050-99(ZZ-1)YY
0-9951-9950-99ZZYY

Examples:
For the current year being ZZXX and a string-expression year of YY:

  • If the current year is 2010, ZZ=20 and XX=10. If YY=59, the resulting year is 1959.
  • If the current year is 2051, ZZ=20 and XX=10. If YY=59, the resulting year is 2051.
  • If the current year is 2010, ZZ=20 and XX=10. If YY=49, the resulting year is 2149.
