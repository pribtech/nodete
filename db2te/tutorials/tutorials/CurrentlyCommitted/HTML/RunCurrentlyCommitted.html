In this page, we will compare the performance of a select statement on a particular row with currently committed turned on, and another select statement on the same row with currently committed turned off, while there is an insert being performed on it.

Run the read statement with cur_comm DISABLED:
Click on the green button in the top right panel. The query we are performing is: 


SELECT * FROM CURRCOMMTUT.CC_TEST WHERE ID = 'test'

View the performance graph:
Click on the 'Metrics Graphed' button below the green button to show the performance graph.

Run the read statement with cur_comm ON:
Next, click on the green button in the bottom right panel. The query we are performing is the same as above.

View the performance graph:
Click on the 'Metrics Graphed' button below the green button to show the performance graph.

Run the write statement that will lock the row:
Click on the second tab, then click on the green button. The query we are performing is: 


UPDATE CURRCOMMTUT.CC_TEST 
       SET TIME='3930-12-18-22.37.17.909000'  
       WHERE ID = 'test'

View the performance graph of the two read statements:
If you click back on the third tab, you will see that the statement with currently committed turned on has no effect, while the one without has a major dip in performance.
