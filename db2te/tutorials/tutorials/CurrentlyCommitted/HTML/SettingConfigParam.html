For new databases created with DB2 version 9.7, the cur_commit configuration parameter is on by default.

Possible values for this parameter are:

  • ON
  • AVAILABLE
  • DISABLED

For upgraded databases, the parameter is set to DISABLED to maintain the same behavior as in previous releases. If you want to use currently committed on cursor stability scans, you need to set the cur_commit configuration parameter to ON after the upgrade.

If you set the configuration parameter to AVAILABLE, you need to explicitly request for currently committed behavior to see the results that are currently committed.

To set the parameter to ON:

  1. find the row whose 'Parameter Name' column is 'cur_commit'
  2. click on the icon under the 'Change' column for that row
  3. enter 'ON', and click on 'OK'
  4. restart DB2
