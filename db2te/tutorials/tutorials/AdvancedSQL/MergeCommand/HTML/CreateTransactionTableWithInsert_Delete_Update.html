This example will use a slightly different form of transaction table. Here the transaction records explicitly tell the MERGE command what to do. 'I' records will insert new values, 'U' records will update existing records, and 'D' will delete records from the product table. If records are not matched, they are ignored. This means that deletes and updates of unknown product records will not take place, but no error record is generated.

This first example will fail because the transaction records contain a duplicate key value (3) for deletes.
