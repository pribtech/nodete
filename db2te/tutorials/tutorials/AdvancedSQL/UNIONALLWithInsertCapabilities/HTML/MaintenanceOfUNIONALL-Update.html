You can also update values within a UNION ALL view, but you cannot make the value in the constraint column move outside the constraint range if you are running a version of DB2 prior to V8 Fixpak 4. This update tries to change a date value outside the range of the constraint. It will not work without an additional clause being used in the creation of the view. For an example of an update statement that succeeds, please refer to the 'UNION Migration' tutorial.
