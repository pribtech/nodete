This SQL will refresh the summary table. Since this summary table was created with the "initially deferred refresh deferred" clause, no data was placed into the summary table. In essence, only the structure of the summary table was created. The REFRESH command will tell DB2 to repopulate this summary table. It will first drop the contents of the table and then create the summary. Since the REFRESH command is not automatic, the command will need to be executed regularly in order the information to be relatively up-to-date. Note that each REFRESH will actually replace the contents of the table, NOT add additional rows.
