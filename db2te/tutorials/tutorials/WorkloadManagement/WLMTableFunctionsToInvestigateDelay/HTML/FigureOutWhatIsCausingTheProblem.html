Now that we know why the application is hung, we can proceed to remedy the situation. We know the application is waiting on a lock. To find out which lock this application is waiting on and which application is holding the lock, we can use the db2pd tool.

First, we need to find out the current transaction number for our hung application. To do so, we need to issue db2pd transactions for the application handle number we got from 2 pages ago.

  1. Open a command line window/terminal
  2. (Windows only): Type db2cmd to open up the CLP
  3. Issue the command (replace X with the application handle #): 
    db2pd -db sample -transactions app=X
    • Look for the TranHdl field.
  4. Keep the CLP or terminal open
