Alternatively, you may also force off 'WLM_tut_app1' by issuing FORCE APPLICATION on application handle A. 

  db2 force application (A)

Another way to diagnose hanging applications due to lock contention is to use the SNAPSHOT_LOCKWAIT monitor table function. This table function provides information on lock holders and waiters. To use this table function, the DFT_MON_LOCK monitor switch configuration parameter must be turned on before the database is started. This switch affects all databases on an instance.
