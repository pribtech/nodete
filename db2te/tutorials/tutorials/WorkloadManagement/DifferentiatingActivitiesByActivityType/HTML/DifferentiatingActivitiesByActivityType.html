Work action sets are used to apply an action to an activity based on what the activity is doing rather than who submitted it (as is done with workloads). This task demonstrates how a work action set can be used to:
  1. collect information about all activities of a certain type
  2. apply a threshold to all activities of a certain type
  3. isolate activities of a certain type by mapping them to a specific service subclass

Actions can be applied to either:

  • all database activities of a certain type (using a database work action set)
  • to only activities of a certain type in a particular service class (using a service class work action set).

This task shows both methods.

There are other actions can be applied, such as collecting statistics for activities of a certain type that are not covered in this task.
