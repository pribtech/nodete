The WLM_GET_WORKLOAD_OCCURRENCE_ACTIVITIES table function is used to get the application handle, unit of work ID, and activity ID of the cursor activity above.

By joining the result of the table function with the APPLICATIONS administrative view, we can find the cursor activity that is run from the TE, whose application name is 'httpd.exe'.

If you click on the 'Result' tab on the bottom panel, you will see two results in the created TE_ACTIVITY view. One for the long-running query, and one for the current query.

The WLM_CANCEL_ACTIVITY stored procedure can be used to cancel the above cursor activity, using the application handle, unit of work ID, and activity ID obtained from the previous step.
