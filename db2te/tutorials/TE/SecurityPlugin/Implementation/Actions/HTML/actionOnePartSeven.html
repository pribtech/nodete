Catching Errors from executeSQL

Steps:

  1. Make the changes to your document as indicated on the right


After the CALL statement has been executed by the JSON server action, one of two things should be done:

  • On success: the panel needs to be reloaded to reflect the absence of the task
  • On failure: an error message should be raised to indicate what went wrong

Two new tasks and a new parameter access type that you have yet to encounter will be required to complete these actions.

The tasks should be straight forward:

  • alert – raises an alert dialog
  • panelReload – reloads the current panel the action is running on and exits the action.

For the local parameter the new attribute value of 'returnObject' allows you to access return data from a given action. Object or array access is done using dot-separated parameters. In this case, we are accessing the actions 'delUser'/'delMembership'/'delGroup' with the return value of 'returnValue'--which is an object in this case--and the value of the string 'STMTMSG'. This may make more sense by looking at the return object of the executeSQL action
