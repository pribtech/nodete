Editing a Table Definition

From this point on in the lab, you will be editing the table definition file '[your_TE_folder]/tableDefinitions/db2AuthLab/db2authCustomMasterView.xml' which you saved earlier in this lab. The name 'table definition' no longer appropriately describes what these files are capable of. They are much more powerful than simply providing a framework to display a table or view. Within the table definition file, you can define countless actions which can be performed on either an entire dataset, or on a row-by-row basis.

Steps:

  1. Open the file 'db2authCustomMasterView.xml' which you saved earlier; a graphical editor will probably work best for this lab. You will be working with the file for the rest of the lab so keep it open.


Throughout the rest of the lab, the table definition will be transformed from its raw state to what is available in the menu today. This will be done by:

  • Editing the table definition file to contain better descriptions of the table and columns within
  • Creating a column to reference a value in that row with another table definition
  • Writing an action to open up a detailed view of authorities and privileges
  • Writing actions to add/delete a user/group/membership
  • Writing an action to change a user's password
  • Creating menus that perform some of the above mentioned tasks

Remember you can view your work anytime by reloading or navigating to the menu item we created earlier in the lab.

If you do not wish to edit the file by hand, you can copy the XML text displayed to the left at almost any time, and overwrite the file. There are some pages where there are lines marked to be deleted; and as such, you will need to manually remove them if you are coping the text. On these pages there will be a note of this at the top of the information in this panel.
