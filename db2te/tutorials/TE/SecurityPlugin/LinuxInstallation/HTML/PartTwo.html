All-in-One Interactive Installation

The plug-in needs to store authentication data in a database. If this database is remote to the DB2 instance that you are configuring this plug-in for, you must do a manual installation. Go on to the next page.

Otherwise, if the authentication database is local, you can do a manual installation, or use the all-in-one interactive installation program, 'DO_IT_ALL_LOCAL.'

Steps:

  1. Make sure you are logged on to the termianl as the DB2 instance owner, and open a bash or ksh shell terminal.

  2. Invoke the script, 'DO_IT_ALL_LOCAL'

  3. Follow the prompts, which will take you through the complete configuration process.

  4. If the program finished successfully, you have finished installing and configuring the plug-in. Otherwise, follow the uninstallation steps in part 6 of this tutorial, and try a manual install.

For further information or trouble-shooting, please refer to the README documentation provided in the plug-in package.
