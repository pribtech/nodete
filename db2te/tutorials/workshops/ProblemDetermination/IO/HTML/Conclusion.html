Conclusion

When you reach a very high number of connections, you should see a value on the Direct Reads indicator that is near the one you write in the configuration file. That is the maximum number of direct reads your system is able to manage. When this maximum is reached, the I/O Intensity will begin to increase. Your system need to delay the requests because it cannot handle more requests at the same time anymore. If the I/O Intensity is reached, your system may become unstable and stop responding.

How to avoid this situation

This kind of problem can be avoided by distributing the load on multiple servers. You achieve that by turning on I/O parallelism on your system if you have multiples I/O devices. For more information on this topic, please refer to I/O server configuration for prefetching and parallelism.

By increasing your bufferpools, you can also reduce the I/O saturation of your system. Bigger bufferpools will result in fewer reads on disks for data that can cached. You can check the Bufferpool Management page for more information.
