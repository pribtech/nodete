Run the query on the right. You should see two returned values.

In this step you will see the values of LOCK_WAITS and LOCK_WAIT_TIME_PERCENT when your system is idle. Because these are snapshot values, you may get a slightly different result when you execute the query at different time.

  • LOCK_WAITS is the number of times that an application has waited for locks.
  • LOCK_WAIT_TIME_PERCENT is the percentage of time spent waiting within the DB2 database server that was spent waiting on locks.

Try executing the query a couple of times to get a rough idea of their idle values.
