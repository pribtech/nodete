This page demonstrates creation of indexes on the first two fact tables to improve query processing by reducing the need for full table scans and repeated sorting of data for queries.

Operation

  1. Create tables 'subscriber_rated_usage_ind' and 'subscriber_dedicated_usage_ind'
  2. Create a regular index on the 'subscriber_msisdn' column as it is used in the join between tables
  3. Create a composite index on the 'subscriber_circle_id' and 'event_start_dt' columns as they are predicates in the WHERE clause
