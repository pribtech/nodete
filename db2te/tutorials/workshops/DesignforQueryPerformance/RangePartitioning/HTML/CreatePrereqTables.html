Note : If the tables are already created, it will indicate so on the console. Proceed to the step no. 3 in this case


Prerequisites

Make sure a multi-partition processing instance and the designdb database are created before proceeding. Please refer to the Design Overview section for detailed instructions.

The prerequisite tables are created in order to run the query against tables created for each feature and compare the query performance improvement in each case, cumulatively

Operation

  1. Click on the action buttons on the Action panel
  2. It create tables enabled with database partitioning
  3. It populate data into the tables. Data is stored in logical data partitions based on the hashing key specified
  4. It creates regular and composite indexes.
