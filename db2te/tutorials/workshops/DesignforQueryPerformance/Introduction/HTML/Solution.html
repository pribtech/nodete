Solution

The layered architecture enables concurrent loading, query, archive and maintenance without compromising query performance.

The idea is to first organize the Data Assets Layer. Then, introduce various performance enhancing database objects in the Performance Layer. Finally, extract data and views based on the business needs in the Business Access Layer.

This workshop introduces a variety of such objects and techniques to attain the goal of efficient physical database design based on the "Best Practices Physical Database Design."

The following design features for efficient database design are showcased:

  1. Database Partitioning
  2. Table Partitioning
  3. MultiDimension Clustering
  4. Row Compression
