Deployment Planning

Planning of deploying DB2 products consist of:

  • Identifying the right DB2 product
  • Selecting the right installation method



DB2 Products

The first step in planning is identifying the right DB2 product. To help you determine which DB2 products to install in your environment, click on the corresponding product link in the below section.
  1. DB2 Server
    2. Several DB2 server products are available depending on your applications, users' functions, and workload characteristics. All DB2 servers are licensed products and some features require additional feature licenses.
    • DB2 Personal Edition
    • DB2 Express-C Edition
    • DB2 Express Edition
    • DB2 Workgroup Server Edition
    • DB2 Connect Enterprise Edition
    • DB2 Enterprise Server Edition

  2. DB2 clients and drivers
    3. Depending on the architecture of your application, you might want to install a DB2 client or driver on each computer that your application is installed on. This setup enables your application to communicate with the DB2 server while the application is running. Understanding the different clients and drivers that are available and selecting the appropriate one is critical for configuring DB2 connectivity and reducing the footprint of the DB2 files.
    • IBM Data Server Runtime Client
    • IBM Data Server Client
    • IBM Data Server Driver for ODBC, CLI, and .NET
    • IBM Data Server Driver

  3. Redistributing clients and drivers
    4. You can freely redistribute all clients and drivers, except the IBM Data Server Client, with your application. You can download these files from the DB2 Application Development Web site at no charge.

The article on the right pane tells you which distributed edition of DB2 is right for you.

Best Practice

Because each server has the full functionality of all clients, including the GUI tools and application development tools, there is no reason to install a client on a computer that already has a server.
