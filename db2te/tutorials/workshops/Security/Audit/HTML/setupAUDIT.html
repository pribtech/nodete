The 'credit_card' table is accessed by different users for different purpose. SECADM Pat wants to audit the employees, managers, and administrators for any activities on that table.

Operation


Auditing can be enabled on different objects like tables, roles, trusted context objects, authorities (SYSADM, DBADM, SECADM) and database.

Pat would like to audit the 'credit_card' table, 'custService' role, SYSADM and DBADM authorities. Based on this requirement, he performs the following:

  • Create audit policies
  • Enable audit on access to 'credit_card' table
  • Enable audit on role 'custService' accessing 'credit_card' table
  • Enable audit on SYSADM and DBADM authorities
  • Archive audit logs and extract information from the archived logs to audit tables
  • Monitor the audit logs for any unauthorized and unusual access
