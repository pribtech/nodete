Least privileges Sam has


You must log into the database as Sam before proceeding with this page.

Sam holds DBADM authority. Check if he can access 'customer' table data and if he can grant/revoke privileges to/from users.

Operation


Sam only has DBADM authority but not ACCESSCTRL and DATAACCESS. Sam executes the SELECT and GRANT statements as shown on the right pane, which should both fail.


Solution


Sam cannot access the 'customer' table. He also cannot grant any privileges to other users. He has given the least privileges to perform only database administrator tasks.
