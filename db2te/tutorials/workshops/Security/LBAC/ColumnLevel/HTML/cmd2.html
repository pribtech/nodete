Grant security label to Pat


You must log into the database as the database creator before proceeding with this page.

In previous page, the security policies and security labels are created. These security labels should be granted to users to provide restricted access to columns of the 'credit_card' table.

Operation


SECADM Pat is granted with 'TOP_SECRET' security label so that he can setup column level LBAC on the 'credit_card' table. Since Pat cannot grant security label to himself, the database creator (who by default holds SECADM) grants him the 'TOP_SECRET' security label.


Solution


Now Pat will be able to setup column level LBAC on the 'credit_card' table.
