Grant security label to Pat


You must log into the database as the database creator before proceeding with this page.

In previous page, security policies and security labels are created. These security labels should be granted to users to provide restricted access to rows of 'dailysales' table.

Operation


SECADM Pat is granted with 'SALES_ORGANIZATION' security label, so that he can setup row level LBAC on 'dailysales' table. Since Pat cannot grant security label to himself, the database creator (who by default holds SECADM) grants him the 'SALES_ORGANIZATION' security label.


Solution


Pat is now able to setup row level LBAC on 'dailysales' table.
